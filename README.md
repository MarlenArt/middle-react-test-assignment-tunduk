# Быстрый запуск
1. Клонирование репозитория:
    ```
    git clone https://github.com/MarlenArt/middle-react-test-assignment-tunduk.git
    cd middle-react-test-assignment
   ```

2. Установка зависимостей:
   В проекте используется React 19 и современный ESLint 9. Рекомендуется использовать флаг для предотвращения конфликтов версий библиотек.    
```npm install --legacy-peer-deps```

3. Запуск в режиме разработки:
```npm run dev``` 
Проект будет доступен по адресу: http://localhost:5173
4. Проверка качества кода:

``` npm run lint  # Линтинг (ESLint + Prettier)```

```npm run test  # Тестирование (Jest + RTL)```

# Стек технологий
1. Frontend: React 19 (Hooks)
2. State Management: Redux Toolkit 
3. Routing: React Router 6 (с синхронизацией состояния в URL)
4. Styling: SCSS Modules (методология BEM-like)
5. Infrastructure: Vite, TypeScript (Strict Mode), ESLint 9 (Flat Config)
6. Notifications: React Toastify

# Принятые решения и архитектура
1. Синхронизация состояния с URL (URL as Single Source of Truth)  
   Все фильтры (имя, вердикт, сортировка) и пагинация синхронизированы с URL-параметрами.  
    Почему: Это позволяет пользователю делиться ссылкой на конкретную выборку данных и корректно использовать кнопки "Назад/Вперед" в браузере.
    Реализация: Кастомный хук useDebounce для поиска, чтобы избежать "дребезга" при записи в URL.

2. Производительность и Мемоизация  
Компоненты: Активно использован React.memo для предотвращения лишних ререндеров тяжелых списков и карточек.  
Вычисления: Массивы для пагинации и трансформации данных (например, split стека технологий) обернуты в useMemo.  
Обработчики: Использован useCallback для функций, передаваемых в дочерние мемоизированные компоненты.
